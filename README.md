# 🦴 Caveman Mode для Harvi Code

> Радикальная экономия токенов. Ноль «воды». Максимальная точность.
[![Harvi Code](https://img.shields.io/badge/Harvi-Code-blue)](https://harvi.pro)
[![Token Savings](https://img.shields.io/badge/token_savings-~75%25-green)](#token-comparison)

---

## ⚡ Что это

**Caveman Mode** — специализированный режим Harvi Code, сокращающий расход токенов на ~75% за счёт телеграфического стиля ответов. Техническая точность сохраняется. «Вода» удаляется.

**Идеально для:**
- 🗂️ Работы с большими кодовыми базами
- 🔄 Длительных сессий рефакторинга
- 💰 Ограниченного бюджета токенов
- ⚡ Быстрых итераций и фиксов

## 🎚️ Уровни экономии

| Уровень | Стиль | Пример |
|---------|-------|--------|
| `lite` | Умеренный. Без приветствий, с сокращениями | `Валидация ввода перед обработкой.` |
| `full` 🔹 | Агрессивный. Без артиклей, фрагменты, паттерн `[объект] [действие] [причина]` | `Null pointer строка 42. Проверить инициализацию.` |
| `ultra` | Экстремальный. ≤3 слова/мысль, символы вместо слов | `Строка 15: x=null. Fix: add check.` |

🔹 `full` — уровень по умолчанию

---

## 🚀 Активация

```bash
/caveman          # full (по умолчанию)
/caveman lite     # умеренная экономия
/caveman ultra    # экстремальная
```

## 📐 Принципы

### Синтаксис
- ❌ Нет: артикли, вводные слова, модальные глаголы, вежливости
- ✅ Да: активный голос, S+V+O, символы (`->` `&` `=` `!=` `?` `!`)

### Формат
- 1 мысль = 1 пункт
- Код без обёрток, если очевиден
- Diff: только изменённые строки

### Иконки вместо мета-текста
| Действие | Иконка |
|----------|--------|
| Чтение | 📖 |
| Запись | 📝 |
| Создание | 🛠️ |
| Удаление | 🗑️ |
| Успех | ✅ |
| Ошибка | ❌ |
| Предупреждение | ⚠️ |

**Пример:**
```
📖 config.toml
🛠️ utils.js: add null check
✅ Done.
```

## 📊 Сравнение токенов: "Что такое турбулентность?"

| Параметр | Стандартный ответ | Caveman (full) |
|----------|------------------|----------------|
| Слова | ~80 | ~20 |
| Токены (оценка) | ~110 | ~28 |
| Структура | Заголовки + списки + формула + примеры | 2 предложения |
| Формула | Развёрнуто: `Re = ρvL/μ` + пояснение | Сжато: `Re > критического` |

**Расчёт экономии:**
```
(110 - 28) / 110 × 100% = ~75% ✅
```

**Стандартный ответ:**
> Турбулентность — это режим течения жидкости или газа, при котором движение частиц становится хаотичным, с образованием вихрей и пульсациями скорости и давления. Ключевые особенности: хаотичное движение частиц по сложным траекториям, образование вихрей разных масштабов, интенсивное перемешивание слоёв, пульсации скорости и давления. Механизм возникновения определяется числом Рейнольдса: Re = ρvL/μ. При превышении критического значения (обычно 2000–4000 для труб) ламинарное течение переходит в турбулентное. Где встречается: атмосферные явления, течение в реках/трубах, аэродинамика транспорта, двигатели, кровеносная система.

**Caveman-ответ:**
> Турбулентность = неупорядоченное движение жидкости/газа с вихрями и пульсациями. Возникает при Re > критического (инерция > вязкость).

✅ Ядро смысла сохранено: определение + механизм + порог Re  
❌ Убрано: примеры применения, детализация признаков, визуальная структура



# Управление правилами экономии токенов

## Структура правил

```
~/.harvi/rules/          # Глобальные (все проекты)
└── caveman-global.md    # Экономия токенов везде

.project/.harvi/rules/   # Только для проекта
└── caveman-local.md     # Экономия токенов в проекте
```

## Подключение

### Глобально (все проекты)

**Windows:**

```cmd
# Создать директорию
if not exist "%USERPROFILE%\.harvi\rules" mkdir "%USERPROFILE%\.harvi\rules"

# Создать файл
echo. > "%USERPROFILE%\.harvi\rules\caveman-global.md"

# Или скопировать готовый
copy caveman-global.md "%USERPROFILE%\.harvi\rules\"
```

**Linux/Mac:**

```bash
# Создать директорию
mkdir -p ~/.harvi/rules

# Создать файл
touch ~/.harvi/rules/caveman-global.md

# Или скопировать готовый
cp caveman-global.md ~/.harvi/rules/
```

### Для проекта

**Windows:**

```cmd
mkdir .harvi\rules
echo. > .harvi\rules\caveman-local.md
```

**Linux/Mac:**

```bash
mkdir -p .harvi/rules
touch .harvi/rules/caveman-local.md
```

## Отключение

### Временно (переименовать)

**Windows:**

```cmd
# Глобальное
ren "%USERPROFILE%\.harvi\rules\caveman-global.md" caveman-global.md.off

# Проектное
ren .harvi\rules\caveman-local.md caveman-local.md.off
```

**Linux/Mac:**

```bash
# Глобальное
mv ~/.harvi/rules/caveman-global.md ~/.harvi/rules/caveman-global.md.off

# Проектное
mv .harvi/rules/caveman-local.md .harvi/rules/caveman-local.md.off
```

### Навсегда (удалить)

**Windows:**

```cmd
# Глобальное
del "%USERPROFILE%\.harvi\rules\caveman-global.md"

# Проектное
del .harvi\rules\caveman-local.md
```

**Linux/Mac:**

```bash
# Глобальное
rm ~/.harvi/rules/caveman-global.md

# Проектное
rm .harvi/rules/caveman-local.md
```

### Отключить все правила

**Windows:**

```cmd
ren "%USERPROFILE%\.harvi\rules" rules.off
```

**Linux/Mac:**

```bash
mv ~/.harvi/rules ~/.harvi/rules.off
```

### Вернуть

**Windows:**

```cmd
ren "%USERPROFILE%\.harvi\rules.off" rules
```

**Linux/Mac:**

```bash
mv ~/.harvi/rules.off ~/.harvi/rules
```

## Приоритет

1. Правила режима — высший приоритет
2. Правила проекта — средний приоритет
3. Глобальные правила — низший приоритет

## Проверка

Правила загружаются при старте сессии. Видны в system prompt:

```
# Rules from ~/.harvi/rules/*.md
# Rules from .harvi/rules-<mode>/*.md
```

## Быстрые команды

**Windows:**

```cmd
# Включить экономию
copy templates\caveman-global.md "%USERPROFILE%\.harvi\rules\"

# Выключить экономию
ren "%USERPROFILE%\.harvi\rules\caveman-global.md" caveman-global.md.off

# Проверить наличие
dir "%USERPROFILE%\.harvi\rules\*.md"
```

**Linux/Mac:**

```bash
# Включить экономию
cp templates/caveman-global.md ~/.harvi/rules/

# Выключить экономию
mv ~/.harvi/rules/caveman-global.md ~/.harvi/rules/caveman-global.md.off

# Проверить наличие
ls ~/.harvi/rules/*.md
```
